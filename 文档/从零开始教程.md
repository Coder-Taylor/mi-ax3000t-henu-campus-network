## ⚡ 基础知识速成——5 分钟看懂网络术语

> 如果你连"IP 地址"是什么都不太清楚，花 5 分钟读完这一节。每个概念只有一句话+一个比喻，够你理解后面的教程。

### 🏠 你的宿舍网络长这样

```
宿舍内部 (LAN)                     外面世界 (WAN/互联网)
                    ┌──────────┐
  你的手机 ─────────→│          │
  你的电脑 ─────────→│  路由器   │────→ 校园网 ──→ 百度/B站/...
  室友手机 ─────────→│          │
                    └──────────┘
                    
  LAN = 局域网        路由器 = "大门"      WAN = 广域网
  (家里)              (内外连接点)         (外面)
```

### 📖 10 个核心术语

| 术语 | 一句话 | 比喻 |
|------|--------|------|
| **IP 地址** | 设备在网络上的"门牌号"，如 `192.168.1.1` | 你家的街道地址 |
| **LAN** | 局域网，宿舍内部网络，IP 一般是 `192.168.x.x` | 你家内部 |
| **WAN** | 广域网，路由器连接外网的出口 | 你家大门外 |
| **网关 (Gateway)** | 访问外网的"出口"，宿舍设备的网关就是路由器 | 小区大门 |
| **DNS** | 把域名翻译成 IP——`baidu.com` → `110.242.68.66` | 电话簿（名字→号码） |
| **DHCP** | 自动给设备发 IP 地址，不用手动设置 | 自动发门牌号 |
| **SSH** | 从电脑远程"进入"路由器命令行的方式（加密的） | 远程桌面，但是文字版 |
| **NAT** | 让宿舍所有设备共享一个校园网 IP 上网 | 公司总机转分机 |
| **路由器** | 一台专用微型电脑，连着内外两个网络 | 你家的门+门卫 |
| **防火墙** | 根据规则拦截/放行网络流量 | 门卫的安检 |
| **子网掩码** | 用 `/24` 或 `/15` 表示网络大小 | "小区围墙"——决定多少个IP算"邻居" |
| **端口 (Port)** | 同一台设备上区分不同服务的编号(0-65535) | 商场里的不同柜台号 |
| **命令行/Terminal** | 用文字控制电脑的方式（黑窗口） | 和电脑"发短信"而不是"点图标" |

### 📶 WiFi 两个模式

```
AP 模式（发射信号）          STA 模式（接收信号）
  ┌──────────┐              ┌──────────┐
  │路由器广播WiFi│            │路由器连校园网│
  │  SSID: xxx │            │ 连 henu-student│
  │  手机来连   │            │  作为上网出口  │
  └──────────┘              └──────────┘
  就像手机开热点              就像手机连WiFi
```

> 💡 **2.4G vs 5G**：2.4G 穿墙好但慢（给你手机用），5G 快但穿墙差（给路由器连校园网用）。

### 💻 什么是"命令行"和"终端"？

> 如果你从未用过命令行，这是最重要的一个概念。**跳过这节 = 后面全看不懂。**

**一句话**：命令行（也叫"终端"、"Shell"）是**用文字控制电脑的方式**。相比用鼠标点来点去，命令行是你**打字**告诉电脑做什么。

```
你平时用电脑（图形界面）：          命令行：
  看到图标 → 鼠标点击                看到光标闪烁 → 输入命令 → 回车
  拖拽文件 → 复制粘贴                输入 `cp file1 file2` → 文件复制完成
```

**在 Windows 上怎么打开命令行？**（三种方法任选）：
1. 按 `Win + R` → 输入 `cmd` → 回车（最经典的黑窗口）
2. 右键开始菜单 → "终端" 或 "Windows PowerShell"
3. 在文件夹地址栏输入 `cmd` → 回车（命令行当前目录=这个文件夹）

打开后你会看到类似这样的东西：
```
C:\Users\你的用户名>
```
这个 `>` 就是**提示符**——表示命令行在等你输入。你输入命令后按回车，电脑执行。

> 💡 **教程中所有 `代码块` 里的内容都是要你在命令行中输入的。** 不要输入 `$` 或 `#` 符号（那是提示符的表示），只输入后面的命令。

### 🗂️ 什么是"路径"和"当前目录"？

**路径**：文件在电脑上的位置，如 `C:\Users\HONOR\Desktop\项目\`。
**当前目录**：命令行"站"在哪个文件夹。输入的命令会在这个文件夹里找文件。

```bash
cd "D:\小米AX3000T刷机教程"    # cd = change directory, 切换目录
#                   ↑ 用双引号包住含中文/空格的路径

dir                           # 列出当前目录下的所有文件
```

> 教程中很多命令需要你先 `cd` 到教程文件夹。如果你把教程解压到了桌面，就先 `cd "C:\Users\你的用户名\Desktop\AX3000T刷机校园网"`。

### 🖥️ OpenWrt 是什么

**一句话**：OpenWrt 是路由器的"操作系统"（就像 Windows 是你电脑的操作系统）。刷 OpenWrt = 给路由器重装系统。

刷完之后你的路由器就变成了一台**完全可控的 Linux 电脑**，可以装软件、写脚本、自动化一切。

### 💾 为什么重启不会丢配置

路由器没有硬盘，用的是 Flash 存储芯片。OpenWrt 用一种叫 **overlayfs** 的"叠层"技术：

```
/rom（出厂固件，只读，永远不变）
  + /overlay（你的修改，可写，重启保留）
    = /（你看到的完整系统）
```

恢复出厂 = 清空 `/overlay`。重启 ≠ 丢配置（因为 overlay 存在 Flash 上）。

### 📇 /etc/hosts 是什么——"你本地的电话簿"

**一句话**：`/etc/hosts` 是一个文件，里面写了"域名 = IP"的对应关系。系统查 DNS 之前**先看这个文件**。

```
/etc/hosts 文件内容：
202.196.96.29   zwyy.henu.edu.cn      ← 告诉系统：zwyy.henu.edu.cn = 202.196.96.29
58.212.123.41   software.henu.edu.cn  ← 告诉系统：software.henu.edu.cn = 58.212.123.41
```

**hosts vs DNS 的区别**：

| | DNS 查询 | /etc/hosts |
|------|----------|------------|
| 像什么 | 打 114 查号台问电话 | 翻自己的通讯录 |
| 速度 | 需要网络请求（几毫秒~几秒） | 本地读取（瞬间） |
| 可靠性 | 可能被劫持、污染、失败 | 100% 可靠（只要文件没写错） |
| 优先级 | **后被查询** | **先被查询**——hosts 优先于 DNS |

**生活比喻**：
- DNS = 打 114 查号台问"河南大学软件学院的电话是多少"
- hosts = 你在手机通讯录里存了"河大软件学院 → 58.212.123.41"

系统先翻通讯录（hosts），找到了就直接用。找不到才打 114（DNS）。

**为什么我们要用它？** 校园网的 DNS 服务器有时对某些域名：
- 返回错误的 IP（打不通）
- 根本不返回结果（"查无此号"）
- 返回的 IP 被 dnsmasq 安全机制拦截（"怕你是诈骗电话"）

这些情况下，直接在 hosts 里写死正确的 IP 是最可靠的办法——**不走 DNS，不受任何干扰**。

> ⚠️ hosts 的缺点：如果网站的 IP 变了，hosts 里的旧 IP 就失效了。但对校园内部网站来说，IP 通常几年不变。

### 🔄 校园网认证是什么

河大校园网 `henu-student` 是**开放 WiFi**（没有 WiFi 密码），但连接后需要**网页认证**才能上外网：

```
连上 henu-student → 打开浏览器 → 自动弹出登录页 → 输学号密码 → 外网通了
```

我们的任务：让路由器自动完成"打开网页→输学号密码"这一步，不需要人操作。

## Part 1：解锁 SSH

### 什么是 SSH？为什么需要解锁？

小米原厂固件是锁死的——你不能用 SSH 远程登录路由器，只能通过米家 App 或 Web 后台管理。**刷机的前提是获得 SSH 访问权限**，相当于"获取路由器的 root 权限"。

我们使用的工具叫 **XMiR-Patcher**（开源项目，GitHub: openwrt-xiaomi/xmir-patcher），它利用小米固件的漏洞来解锁 SSH。

### ⚠️ 开始前的准备

**首先确认你的路由器状态**：

| 路由器状态 | 你需要做什么 |
|-----------|-------------|
| **全新，刚拆箱** | 通电，网线连电脑，浏览器打开 `miwifi.com`。按小米的引导设置管理密码（**记住这个密码！**后面用得到），完成初始化。然后继续 Part 1 |
| **之前用过，记得管理密码** | 通电，网线连电脑，确认浏览器能登录 `miwifi.com`。继续 Part 1 |
| **之前用过，忘了管理密码** | 通电，用牙签按住 Reset 孔 10 秒以上，等指示灯闪烁 → 路由器恢复出厂。然后按"全新"的情况重新初始化 |

> **路由器要不要插 WAN 口（连校园网/外网）？**
> XMiR-Patcher 只需要电脑和路由器在同一局域网就能工作——**不需要**路由器能上网。所以 WAN 口留空，只需要一根网线连接电脑和路由器 LAN4 口。

### 操作步骤

#### 步骤 1.1：物理连接

```
  电脑                  AX3000T 路由器
  ┌──────┐             ┌──────────────┐
  │      │   网线接    │              │
  │ 网口 ├────────────→│ LAN4 口       │  ← 插 LAN4，不是 WAN！
  │      │             │              │
  └──────┘             └──────┬───────┘
                              │ 插电源（WAN 口留空）
                              │
                              │ 等 2 分钟，指示灯变白/蓝常亮
                              │
                              │ 浏览器打开 miwifi.com
                              │ 确认能看到小米管理页面
```

逐步确认：
1. 网线一头插电脑网口，一头插路由器 **LAN4 口**（四个 LAN 口中最右边那个）
2. 路由器通电，等待约 2 分钟
3. 电脑浏览器打开 `http://192.168.31.1` 或 `http://miwifi.com`
4. 你应该看到小米路由器的管理登录页
   - **如果你是第一次用**：会看到初始化向导，跟着设置管理密码。设完后用这个密码登录
   - **如果你之前用过**：输入你设的管理密码登录
5. 确认能进入管理后台（能看到"路由状态"等信息），说明连接正常

> **如果打不开 miwifi.com？**
> - 检查网线是否插在 LAN4 口（不是 WAN 口！WAN 是蓝色标记的那个）
> - 检查电脑是否开启了 WiFi（如果连着别的 WiFi，浏览器可能优先走 WiFi 而不是网线）→ 暂时关掉 WiFi
> - 打开 Windows 命令行，输入 `ipconfig`，看"以太网适配器"的 IPv4 是否以 `192.168.31.x` 开头。如果不是，说明电脑没从路由器拿到 IP → 检查网线

#### 步骤 1.2：运行 XMiR-Patcher

先在文件资源管理器中进入教程文件夹：
```
小米AX3000T刷机教程openwrt大分区版/
└── step1解锁ssh/
    └── xmir-patcher-main/
        ├── !START.bat          ← 双击这个运行
        ├── python/             (内置 Python，不需要你自己装！)
        ├── menu.py             (菜单程序)
        ├── create_backup.py    (备份脚本)
        ├── install_ssh.py      (解锁 SSH 脚本)
        ├── install_bl.py       (刷 U-Boot 脚本)
        └── ...
```

**详细操作**：

**第 1 步**：双击 `!START.bat`

会弹出一个**命令行黑窗口**（不要怕！这就是前面讲的"命令行"），显示类似：
```
============================================
  XMiR-Patcher - Xiaomi Router Toolkit
============================================
[1] Connect to router
[2] Read router info
[3] Create backup (IMPORTANT!)
[4] Unlock SSH
[5] Install uboot (大分区)
[6] Set uboot env variables
[7] Install firmware
[0] Exit
============================================
Please select an option:
```
（你的版本菜单文字可能略有不同，但功能一样）

**第 2 步**：输入 `1` 回车 → Connect to router

工具会提示你输入路由器的 IP 和管理密码：
```
Router IP [192.168.31.1]:        ← 直接回车（用默认的）
Router password:                 ← 输入你在 miwifi.com 设的管理密码
```
输入密码时**屏幕不会显示任何字符**（不显示 `***`），这是正常的！输入完直接回车。

如果连接成功，会显示类似 `Connected successfully` 或获取到了 STOK（一个令牌）。

> **如果连接失败**：检查密码是否正确（去 miwifi.com 重新登录确认），确认路由器 IP 是 `192.168.31.1`。

**第 3 步**：输入 `2` 回车 → Read router info

读取路由器型号、固件版本等信息。会显示类似：
```
Model: Xiaomi Mi Router AX3000T
Firmware: 1.0.xx
...
```
确认显示的是 AX3000T，不是别的型号。

**第 4 步**：输入 `3` 回车 → ⚠️ **Create backup（极其重要！）**

这一步会备份路由器的全部分区到电脑的 `backups/` 目录。**必须做！**
- 执行时间：约 2-5 分钟（正在读取整个 128MB Flash）
- 结束后在 `xmir-patcher-main/backups/` 目录下会出现很多 `.bin` 文件
- **马上把这些文件复制到网盘/U 盘/另一个地方！** 丢了就没了

**第 5 步**：输入 `4` 回车 → Unlock SSH

工具会修改小米的启动脚本，让路由器开机时启动 dropbear（SSH 服务器）。
- 执行完成后路由器可能会自动重启
- 等 1-2 分钟路由器重新启动完成

**第 6 步**：输入 `5` 回车 → Install uboot

刷入大分区版 U-Boot（替换小米原厂的）。这是关键一步——大分区 U-Boot 会重新划分 Flash 空间，给 overlay 分配 78MB（原厂只给 ~10MB）。
- 执行很快（几秒到几十秒）

**第 7 步**：输入 `6` 回车 → Set env variables

设置 U-Boot 环境变量，告诉 U-Boot "从新的大分区启动"。

**第 8 步**：输入 `0` 退出工具。

#### 步骤 1.3：确认 SSH 已解锁

打开一个**新的命令行窗口**（Win+R → `cmd`），输入：

```bash
ssh root@192.168.31.1
```

**第一次连接会看到**：
```
The authenticity of host '192.168.31.1' can't be established.
...
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```
输入 `yes` 回车（这是 SSH 的安全确认，只出现一次）

如果出现 `password:` 的提示或直接进入了命令行（显示 `root@XiaoQiang:~#`），说明 SSH 已经通了！

> **如果 SSH 连不上**：
> 1. 确认路由器已经重启完成（等 2 分钟）
> 2. 确认电脑和路由器还连着网线
> 3. 在 XMiR-Patcher 中重新执行 [4] Unlock SSH
> 4. Windows 可能需要先启用 OpenSSH 客户端：设置 → 应用 → 可选功能 → 添加功能 → OpenSSH 客户端

**默认 SSH 密码**：解锁后 root 密码通常是 `root` 或者为空（直接回车），也可能是你在小米 Web 后台设的管理密码。XMiR-Patcher 通常会把密码设为空。

<a name="part2"></a>
## Part 2：刷入 ImmortalWrt

### 什么是 ImmortalWrt？

ImmortalWrt 是 OpenWrt 的一个中国社区维护版本。相比原生 OpenWrt，它：
- 预装中文语言包
- 预配国内软件镜像源（更新/装包更快）
- 对中国用户的网络环境做了优化

我们使用的是 **大分区版**——把原来小米切得很碎的分区合并成一个大分区，让你有 78MB 可用空间（原版只有 10MB 左右）。

> **大分区版 vs 普通版**：前面 Part 1 我们刷了"大分区 U-Boot"，这一步刷的固件必须也是"大分区版固件"——两者必须匹配。`step2刷入openwrt/` 里的固件已经是对应的大分区版本。

### 操作步骤

#### 步骤 2.1：找到固件文件

在文件资源管理器中确认固件文件存在：
```
小米AX3000T刷机教程openwrt大分区版/
└── step2刷入openwrt/

#### 步骤 2.5：刷入固件

**在路由器的 SSH 窗口中**执行（⚠️ 这是最关键的一步）：

```bash
sysupgrade -n -v /tmp/immortalwrt-24.10.5-*-sysupgrade.bin
#  ↑          ↑  ↑          ↑
#  │          │  │          └── 固件文件路径（* 是通配符，匹配任意字符）
#  │          │  └── 详细输出模式，显示刷机进度
#  │          └── 不保留原配置（干净安装）
#  └── OpenWrt 的刷机命令
```

**参数详解**：
- `sysupgrade` = System Upgrade，OpenWrt 的标准刷机工具
- `-n` = 不保留旧配置，干净安装（刷机时必加）
- `-v` = verbose，显示详细进度
- 路径中的 `*` 是通配符（wildcard），避免你输入完整的 30+ 字符文件名

**按下回车后，屏幕会显示类似**：
```
Saving config files...
Commencing upgrade. Closing all shell sessions.
Connection to 192.168.31.1 closed by remote host.
```
然后你的 SSH 连接会断开——**这是正常的！** 路由器正在重启刷写新固件。

**刷机过程中路由器的状态**：
```
时间轴：
  t=0s    按下回车 → SSH 断开
  t=5s    路由器开始刷写 Flash
  t=30s   刷写完成，自动重启
  t=30-60s 第一次重启（指示灯闪烁/灭）  ← 不要断电！
  t=60s   系统启动中
  t=120s  完全启动，指示灯稳定常亮
```

**⚠️ 这 2-3 分钟内最重要的是：**
1. **不要断电！**
2. **不要拔网线！**
3. **不要按 Reset！**
4. 安静等待，可以去喝杯水 😊

> **如果断电了会怎样？** 刷机写的是 mtd8(mtd9) 分区，断电需要重刷。只要不损坏 BL2/FIP（mtd1/mtd4），路由器就能通过 U-Boot 恢复模式救回来。所以**不要慌**，但要尽量避免。

等 3-5 分钟（保守点），然后验证刷机成功：

**验证 1——浏览器访问 Web 管理**：
1. 浏览器打开 `http://192.168.1.1`
   - ⚠️ 注意：是 `192.168.1.1`（**不是** `192.168.31.1`！刷机后 IP 变了）
2. 你应该看到 **LuCI** = ImmortalWrt 的 Web 管理界面（中文，深色/浅色主题）
3. 用户名：`root`，密码：**直接点"登录"**（空密码）

**验证 2——SSH 连接**：
```bash
ssh root@192.168.1.1
# 密码为空，直接回车
```

**验证 3——确认大分区成功**（在 SSH 中）：
```bash

```
输出应该显示：
```
Filesystem            Size      Used Available Use% Mounted on
ubi0_1               58.0M      1.2M     56.8M   2% /overlay
```
看到 `58.0M`（约 58MB 可用）说明大分区刷成功了！如果只有 5-10M，说明没刷成大分区。

**验证 4——确认系统版本**：
```bash

```
输出：
```
NAME="ImmortalWrt"
VERSION="24.10.5"

```

### 3.0 Linux 是什么？为什么你的路由器跑的是它？

**一句话**：Linux 是一个操作系统，就像 Windows 是你电脑的操作系统。但 Windows 有桌面、有图标、有鼠标，Linux 可以**只有命令行**。

```
你的电脑                              你的路由器
┌─────────────────────┐             ┌──────────────────┐
│ Windows 10/11        │             │ Linux (OpenWrt)   │
│ 有桌面、鼠标、窗口    │             │ 只有命令行         │
│ 操作：点击图标        │             │ 操作：输入命令      │
│ 硬盘 256GB+          │             │ Flash 128MB       │
│ RAM 8GB+             │             │ RAM 256MB         │
│ 有显卡、声卡、屏幕    │             │ 无显卡、无屏幕      │
└─────────────────────┘             └──────────────────┘
```

**为什么路由器用 Linux？**
- Linux 免费、开源、可定制——厂商可以只装需要的部分
- Linux 网络功能极其强大（整个互联网的服务器大多跑 Linux）
- 省资源——不需要图形界面，128MB 内存就够
- 路由器本质上就是一台"专职转发网络数据的微型电脑"

**OpenWrt 是什么？** OpenWrt = **专门给路由器用的 Linux 发行版**。它和普通 Linux 的关系就像：
- Windows 10/11 = 给台式机/笔记本用的操作系统
- iOS/Android = 给手机用的操作系统
- OpenWrt = 给路由器用的操作系统

**OpenWrt 和普通 Linux 的区别**：
| 特性 | 普通 Linux（如 Ubuntu） | OpenWrt |
|------|------------------------|---------|
| 用途 | 桌面/服务器 | 路由器 |
| 图形界面 | 有（GNOME/KDE） | Web 管理页（LuCI） |
| 包管理器 | apt/dnf/pacman | opkg |
| 网络配置 | 分散在多个文件 | UCI 统一管理 |
| 命令集 | GNU coreutils（完整） | BusyBox（精简嵌入式版） |
| 安装大小 | 5-20 GB | ~10 MB（压缩后） |

> 💡 **关键理解**：你 SSH 进去之后看到的黑窗口，就是一个**真正的 Linux Shell**。你在上面执行的 `ls`、`cat`、`ping` 都是标准的 Linux 命令。学会操作 OpenWrt = 学会了 Linux 命令行基础。

---

### 3.1 root 是什么？为什么叫"root"？

**一句话**：`root` 是 Linux 的**超级管理员账户**——拥有最高权限，能做任何事。

```
Windows 权限模型：                  Linux 权限模型：
  你的账户（管理员）                     root（超级管理员，UID=0）
    ↓ 能装软件、改设置                    ↓ 能做任何事——改系统文件、删分区...
  其他用户（标准账户）                   普通用户（UID≥1000）
    ↓ 不能装软件                          ↓ 只能操作自己的文件
```

**为什么你登录的是 root？** 路由器只有一个用户——root。因为：
1. 路由器不需要多用户（不像服务器）
2. 所有配置都是系统级别的（改网络、改 WiFi、装软件）——只有 root 能做
3. SSH 命令行中 `#` 就是 root 的标志（普通用户是 `$`）

```bash
root@ImmortalWrt:~#    ← 这是 root，以 # 结尾
user@ImmortalWrt:~$    ← 这是普通用户，以 $ 结尾
```

### 3.3 用 SSH 进入路由器的"内部"

**第一次进入后你会看到什么？**

### 3.4 Linux 命令行速成——5 个你必须会的命令

以下 5 个命令会在后面教程反复出现。不需要背——先扫一眼，遇到时回来看。

| 命令 | 全称 | 做什么 | 例子 |
|------|------|--------|------|
| `ls` | list | 列出目录下的文件 | `ls /etc/config/` |
| `cat` | concatenate | 显示文件内容 | `cat /etc/os-release` |
| `cd` | change directory | 切换当前目录 | `cd /etc/config/` |
| `cp` | copy | 复制文件 | `cp file1 file2` |
| `vi` / `vim` | visual editor | 编辑文件（文本编辑器） | `vi /etc/config/network` |

**Linux 路径 vs Windows 路径**：
```
Windows:  C:\Users\你的名字\Desktop\项目\
Linux:    /etc/config/network
          ↑ 正斜杠 /    ↑ 没有盘符(C:)  ↑ 区分大小写！
```

> ⚠️ Linux 严格区分大小写！`Network` ≠ `network` ≠ `NETWORK`。

**vi 编辑器生存指南**：vi 是 Linux 默认的文本编辑器，但操作方式和 Windows 的记事本完全不同。如果你从未用过：
```bash
# 打开文件
vi /etc/config/network

# vi 有两种模式：
# 1. "命令模式"（刚进去时的状态）——键盘按键是命令，不会输入文字
# 2. "编辑模式"（按 i 进入）——可以正常输入文字

按 i        → 进入编辑模式（左下角显示 -- INSERT --）
修改内容
按 Esc      → 退出编辑模式，回到命令模式
输入 :wq    → 保存并退出（write + quit）
输入 :q!    → 不保存退出（quit + force）
```

> 💡 **提示**：如果觉得 vi 太难用，可以先用 LuCI Web 界面做配置。学 vi 可以以后再说——它不影响完成教程。

## Part 4：安全加固

> **现状**：刷完的系统 WiFi 无密码、root 无密码——任何人连上你的路由器都能完全控制它。

### 4.1 设置 root 密码

**方法 A：通过 Web 界面**

系统 → 管理权 → 输入新密码 → 保存并应用

**方法 B：通过 SSH**

```bash
ssh root@192.168.1.1
passwd
# 输入新密码，回车
# 再次输入，回车
```

> 💡 **密码建议**：至少 8 位，包含字母、数字、特殊符号。如 `MyPass123#`
> 
> 设置后，Web 登录和 SSH 登录都需要这个密码。

### 4.2 设置 WiFi 密码

**方法 A：通过 Web 界面**

网络 → 无线 → 找到 2.4G 和 5G 的接口 → 编辑 → 无线安全 → 选 WPA2-PSK → 输入密码 → 保存并应用

**方法 B：通过 SSH**

```bash
# 设置 2.4G WiFi
uci set wireless.default_radio0.ssid='你的WiFi名'
uci set wireless.default_radio0.encryption='psk2'
uci set wireless.default_radio0.key='你的WiFi密码'

# 设置 5G WiFi
uci set wireless.default_radio1.ssid='你的WiFi名'
uci set wireless.default_radio1.encryption='psk2'
uci set wireless.default_radio1.key='你的WiFi密码'

# 保存并立即生效
uci commit wireless
wifi reload
```

参数说明：
- `psk2` = WPA2-PSK（目前最常用的 WiFi 加密方式，兼容性好）
- `sae` = WPA3（新一代加密，更安全但老设备可能不支持）

### 4.3 配置 SSH 密钥免密登录（可选但推荐）

每次 SSH 都输密码既麻烦又不够安全。SSH 密钥用"非对称加密"实现免密登录：

```bash
# 在你的 Windows 电脑上（不是路由器上）：
# 1. 生成密钥对
ssh-keygen -t ed25519 -f ~/.ssh/ax3000t_rsa -N ""

# 2. 把公钥上传到路由器
scp ~/.ssh/ax3000t_rsa.pub root@192.168.1.1:/tmp/

# 3. SSH 到路由器，添加公钥
ssh root@192.168.1.1
mkdir -p /etc/dropbear
cat /tmp/ax3000t_rsa.pub >> /etc/dropbear/authorized_keys
chmod 600 /etc/dropbear/authorized_keys

# 4. 之后免密登录
ssh -i ~/.ssh/ax3000t_rsa root@192.168.1.1
```

> 💡 创建 SSH config 文件 `~/.ssh/config`：
> ```
> Host ax3000t
>     HostName 192.168.1.1
>     User root
>     IdentityFile ~/.ssh/ax3000t_rsa
> ```
> 之后只需输入 `ssh ax3000t` 就能连接。

<a name="part5"></a>
## Part 5：WiFi WAN——让路由器无线连校园网

这是整个项目**最核心的一步**。

### 前置知识：AP vs STA

```
AP 模式（Access Point）        STA 模式（Station）
   你连 WiFi 热点的模式           WiFi 设备连路由器的模式
   ┌──────────┐                 ┌──────────┐
   │路由器发射WiFi│               │路由器接收WiFi│
   │让别人连      │               │去连校园网    │
   └──────────┘                 └──────────┘
```

默认情况下，两个 radio 都是 AP 模式。我们需要把其中一个改成 STA 模式去连接校园网。

### 5.1 安装必要软件

虽然 `wpad-openssl`（包含 STA 客户端功能）通常预装了，但确认一下：

```bash
ssh root@192.168.1.1
opkg update
opkg list-installed | grep wpad
```

如果输出里有 `wpad-openssl`，就说明已经有了。

### 5.2 扫描校园网 WiFi

```bash
# 先看有什么 WiFi 信号
iw dev phy0-ap0 scan | grep -E "SSID|signal" | head -20
```

> **⚠️ 注意**：`iw scan` 在某些路由器上可能导致 WiFi 短暂断连。如果扫描后 SSH 断开，等 30 秒再重连即可。只扫一次，不要反复扫。

你会看到类似输出：
```
SSID: henu-student
signal: -56.00 dBm
SSID: henu-student
signal: -63.00 dBm
...
```

记下信号最好的那个。dBm 越接近 0 越好。

### 5.3 决定用哪个 radio 连接校园网

| radio | 频段 | 建议用途 | 理由 |
|-------|------|----------|------|
| radio0 | 2.4G | 宿舍 AP（发射热点） | 穿墙好，覆盖广 |
| radio1 | 5G | WAN（连接校园网） | 速度快，干扰少 |

### 5.4 配置 STA 模式

备份原配置：
```bash
cp /etc/config/wireless /etc/config/wireless.bak
cp /etc/config/network /etc/config/network.bak
cp /etc/config/firewall /etc/config/firewall.bak
```

配置 radio1 为 STA 模式：
```bash
# 改为 STA 客户端模式
uci set wireless.default_radio1.mode='sta'
uci set wireless.default_radio1.ssid='henu-student'
uci set wireless.default_radio1.encryption='none'
uci delete wireless.default_radio1.key 2>/dev/null
uci set wireless.default_radio1.network='wwan'
uci set wireless.radio1.channel='auto'
uci commit wireless
```

> 如果你的校园网 WiFi 有密码（WPA-Enterprise 等），请参阅附录 B 的特殊情况处理。

### 5.5 创建 wwan 网络接口

```bash
uci set network.wwan='interface'
uci set network.wwan.proto='dhcp'
uci commit network
```

### 5.6 配置防火墙

```bash
uci add_list firewall.@zone[1].network='wwan'
uci commit firewall
```

### 5.7 应用并验证

```bash
wifi reload
sleep 5

# 检查 STA 连接状态
iw dev phy1-sta0 link

# 让它获取 IP
ifup wwan
sleep 3

# 看看拿到了什么 IP
ip addr show dev phy1-sta0 | grep inet
```

如果输出里有 `inet 10.x.x.x`，说明从校园网 DHCP 服务器拿到了 IP。

网络测试：
```bash
# 测试校园网内网（应该通）
ping -c 2 172.29.35.27    # 认证服务器

# 测试外网（此时应该不通，因为还没认证）
ping -c 2 8.8.8.8
```

> **此时的状态**：路由器已经通过 WiFi 连上了校园网，拿到了内网 IP，能访问校园内网，但上不了外网。这就像你用手机连上 henu-student 但还没打开浏览器认证一样。

<a name="part6"></a>
## Part 6：部署校园网认证脚本

### 6.1 理解校园网认证流程

河大校园网使用 **Portal 认证**：

```
连上 henu-student → 拿到 IP → 仅能访问校园内网
                                    ↓
                           打开浏览器任意网页
                                    ↓
                       被强制跳转到 Portal 页面
                       (172.29.35.36:6060)
                                    ↓
                         输入学号密码点登录
                                    ↓
                            背后执行三步认证
                                    ↓
                            认证通过，外网通了
```

我们的脚本就是模拟浏览器执行这三步 HTTP 请求，去掉"手动打开网页"的环节。

### 6.2 创建认证脚本

SSH 到路由器，执行：

```bash
mkdir -p /etc/campus_network
```

然后把下面的脚本**完整复制**到 `/etc/campus_network/auto_login.sh`。

> **你需要修改的地方**：第 4-7 行的 `USERNAME`、`PASSWORD`、`OPERATOR_SUFFIX`、`CAMPUS_CODE` 改成你自己的。

```bash
cat > /etc/campus_network/auto_login.sh << 'EOF'
#!/bin/sh

# ====== 改成你自己的信息 ======
USERNAME="你的学号"
PASSWORD="校园网密码"
OPERATOR_SUFFIX="@henuyd"    # 移动: @henuyd, 联通: @henult, 电信: @henudx
CAMPUS_CODE="你的校编码"
# ============================

LOG_FILE="/tmp/campus_network.log"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" >> $LOG_FILE
}

# 获取网络参数（WAN IP、时间戳、UUID）
get_auth_params() {
    WAN_IP=$(ip route get 8.8.8.8 2>/dev/null | awk '{for(i=1;i<=NF;i++) if($i=="src") print $(i+1)}')
    TIMESTAMP=$(($(date +%s) * 1000))
    UUID=$(cat /proc/sys/kernel/random/uuid 2>/dev/null)
    echo "$WAN_IP $TIMESTAMP $UUID"
}

# 第一步：验证账号密码
first_auth() {
    log "第一步认证..."
    RESPONSE1=$(wget -q -O - \
        --post-data="campusCode=${CAMPUS_CODE}&username=${USERNAME}&password=${PASSWORD}&operatorSuffix=${OPERATOR_SUFFIX}" \
        --header="Content-Type: application/x-www-form-urlencoded" \
        --header="Referer: http://172.29.35.36:6060/" \
        "http://172.29.35.27:8088/aaa-auth/api/v1/auth" 2>&1)
    
    log "第一步响应: $RESPONSE1"
    if echo "$RESPONSE1" | grep -q '"code":1'; then
        return 0
    else
        return 1
    fi
}

# 第二步：二次校验
second_auth() {
    log "第二步认证..."
    RESPONSE2=$(wget -q -O - \
        --post-data="username=${USERNAME}&password=${PASSWORD}&operatorSuffix=${OPERATOR_SUFFIX}" \
        --header="Content-Type: application/x-www-form-urlencoded" \
        --header="Referer: http://172.29.35.36:6060/" \
        "http://172.29.35.27:8882/user/check-only" 2>&1)
    
    log "第二步响应: $RESPONSE2"
    if echo "$RESPONSE2" | grep -q '"code":1'; then
        return 0
    else
        return 1
    fi
}

# 第三步：门户最终认证
portal_auth() {
    log "第三步门户认证..."
    
    PARAMS=$(get_auth_params)
    WAN_IP=$(echo "$PARAMS" | cut -d' ' -f1)
    TIMESTAMP=$(echo "$PARAMS" | cut -d' ' -f2)
    UUID=$(echo "$PARAMS" | cut -d' ' -f3)
    
    log "使用参数: IP=$WAN_IP, TS=$TIMESTAMP, UUID=$UUID"
    
    RESPONSE3=$(wget -q -O - \
        --header="Referer: http://172.29.35.36:6060/" \
        --header="Cookie: macAuth=; ABMS=362ee66b-fa1f-4ef9-a651-bfd9d61d194a" \
        -T 10 \
        "http://172.29.35.36:6060/quickauth.do?userid=${USERNAME}%40henuyd&passwd=${PASSWORD}&wlanuserip=${WAN_IP}&wlanacname=HD-SuShe-ME60&wlanacIp=172.22.254.253&timestamp=${TIMESTAMP}&uuid=${UUID}" 2>&1)
    
    log "第三步响应: $RESPONSE3"
    
    if echo "$RESPONSE3" | grep -q '"message":"认证成功"'; then
        log "✅ 第三步认证成功"
        return 0
    else
        log "❌ 第三步认证失败"
        return 1
    fi
}

# 网络检测
check_network() {
    ping -c 2 -W 3 8.8.8.8 > /dev/null 2>&1
}

# 完整认证流程
full_auth() {
    log "开始完整认证流程..."
    
    if first_auth; then
        sleep 2
        if second_auth; then
            sleep 2
            if portal_auth; then
                log "🎉 所有认证步骤成功完成"
                return 0
            else
                log "❌ 第三步认证失败"
                return 1
            fi
        else
            log "❌ 第二步认证失败"
            return 1
        fi
    else
        log "❌ 第一步认证失败"
        return 1
    fi
}

main() {
    log "=== 校园网认证启动 ==="
    
    if check_network; then
        log "网络已连通，无需认证"
        return 0
    fi
    
    log "网络未连通，开始完整认证流程..."
    if full_auth; then
        sleep 5
        if check_network; then
            log "🎉 认证成功，网络已连通！"
        else
            log "⚠️ 认证完成但网络检测失败"
        fi
    else
        log "❌ 认证流程失败"
    fi
}

main
EOF

# 给脚本执行权限
chmod +x /etc/campus_network/auto_login.sh
```

### 6.3 手动测试

```bash
# 运行脚本
/etc/campus_network/auto_login.sh

# 查看日志
cat /tmp/campus_network.log
```

如果最后一行是 `🎉 认证成功，网络已连通！`，恭喜！

验证外网：
```bash
ping 8.8.8.8
```

如果 ping 通，说明一切正常。

### 6.4 如果认证失败了

```bash
cat /tmp/campus_network.log
```

看哪一步失败：

| 错误 | 可能原因 | 解决方法 |
|------|----------|----------|
| 第一步 code≠1 | 账号密码/运营商后缀/校编码错误 | 检查 4 个参数 |
| 第一步连不上 | 校园网没连上 | 回 Part 5 检查 STA 状态 |
| 第二步 code≠1 | 账号类型不匹配 | 确认运营商后缀 |
| 第三步 message≠认证成功 | IP/时间戳/UUID/AC参数出错 | 检查 `wlanacname` 和 `wlanacIp` 是否正确 |

<a name="part7"></a>
## Part 7：定时重连

### 7.1 设置 cron

校园网认证有时效（可能几小时到一天）。我们需要路由器定期检查，断了就自动重新认证。

### 7.1 理解断网的不同原因

并不是所有"网断了"都要重新认证！不同的断网原因需要不同的处理：

| 原因 | 表现 | 正确处理 | 耗时 |
|------|------|----------|:---:|
| DHCP 租约问题 | WiFi 连着，IP 丢了 | DHCP 续租 | 2-3秒 |
| **设备重连断联** ⚠️ | 某设备离开半天后重连→其他设备全断 | DHCP 租约 168h + hostapd 调优 | Phase 13 已修复 |
| WiFi 掉了 | STA 断开 | 重连 STA | 3-5秒 |
| Portal 过期 | WiFi+IP 都有，但 ping 不通外网 | 重新认证 | 10-15秒 |

> 守护进程（`fast_recovery_daemon`）会**自动判断**原因并用最快方式恢复，秒级完成。

### 7.2 设置认证定时检测

守护进程处理路由层面的恢复（快），cron 定时任务兜底认证过期（慢但稳）：

```bash
# 添加定时任务：每 5 分钟运行一次认证脚本
echo "*/5 * * * * /etc/campus_network/auto_login.sh" >> /etc/crontabs/root

# 重新连接后 30 秒立即认证
echo "@reboot sleep 30; /etc/campus_network/auto_login.sh" >> /etc/crontabs/root

# 重启 cron 服务使配置生效
/etc/init.d/cron restart
```

> **为什么是 5 分钟？**
> - Portal 认证有效期通常几小时，5 分钟检查一次完全够用
> - 之前 3 分钟太频繁，没必要地频繁访问认证服务器
> - 1 分钟就更过了——对认证服务器可能构成压力
> - 守护进程每秒检查路由连通性（秒级恢复），cron 只兜底认证过期（分钟级）

### 7.2 验证 cron 是否工作

等 5-6 分钟，然后：
```bash
cat /tmp/campus_network.log
```

你应该看到类似：
```
[2026-06-04 06:30:00] === 校园网认证启动 ===
[2026-06-04 06:30:01] 网络已连通，无需认证
```

### 7.3 NAT 修复（重要！）

在某些 OpenWrt 24.10 版本中，NAT 可能不生效——路由器自己能上网，但连着的设备不行。

如果发现这个问题：
```bash
# 关闭 fullcone NAT
uci set firewall.@defaults[0].fullcone='0'
uci commit firewall
fw4 restart
```

## 附录 A：背后的原理

### A.1 刷机到底改了什么？

```
小米原厂固件                          刷之后
┌──────────────────┐              ┌──────────────────┐
│ 小米定制 Linux    │              │ ImmortalWrt      │
│ (基于 OpenWrt 旧版)│    ──→      │ (OpenWrt 24.10)  │
│                  │              │                  │
│ 限制：            │              │ 限制：            │
│ - SSH 锁死       │              │ - ✅ 完全开放     │
│ - 不能装任意软件  │              │ - ✅ 几千个软件包  │
│ - 分区碎片化      │              │ - ✅ 大分区 78MB  │
│ - 小米后台控制    │              │ - ✅ 你完全控制    │
└──────────────────┘              └──────────────────┘
```

### A.2 路由器是怎么启动的？

> 理解启动过程对调试和救砖非常重要。不用担心看不懂——每个阶段后面都有小白解释。

```
    通电
      │
      ▼
┌──────────┐
│  BL2      │  ← Boot Loader Stage 2
│  (ROM里)  │     芯片出厂烧死在 ROM 里的启动代码，无法修改
│           │     就像电脑的 BIOS，是"天生"的
│           │     职责：初始化内存 → 从 Flash 加载下一阶段
└────┬─────┘
     │
     ▼
┌──────────┐
│  FIP      │  ← Firmware Image Package（固件映像包）
│  (Flash)  │     包含两个东西：
│           │     • ATF (ARM Trusted Firmware) — 配置安全环境
│           │     • U-Boot — 引导加载程序，相当于电脑的 GRUB
└────┬─────┘
     │  U-Boot 的职责：
     │  • 初始化网口、串口等硬件
     │  • 从 ubi_kernel 分区加载 Linux 内核到内存
     │  • 传递内核启动参数
     │  • 如果按住 Reset 键通电 → 进入恢复模式（救砖用）
     ▼
┌──────────┐
│  Linux    │  ← 内核启动
│  Kernel   │     初始化所有硬件驱动（WiFi、网卡、闪存...）
│           │     挂载 squashfs 固件到 /rom（只读）
│           │     挂载 ubifs 用户数据到 /overlay（可写）
│           │     用 overlayfs 叠成完整的根文件系统 /
│           │     最后启动第一个用户态进程：init
└────┬─────┘
     │
     ▼
┌──────────┐
│  init     │  ← 第一个用户态进程 (PID=1)
│  (procd)  │     按优先级依次启动所有服务：
│           │     network → firewall → dnsmasq → cron → LuCI...
│           │     我们的 campus_auth 和 fast_recovery 也在这时启动
└────┬─────┘
     │
     ▼
 🟢 正常运行，WiFi 亮起
```

**小白理解**：路由器启动就像盖房子——
- BL2 = 打地基（出厂就有了）
- FIP/U-Boot = 搭框架（决定房子怎么盖）
- Linux Kernel = 通水电（让硬件能工作）
- init = 搬家具进去（启动各种服务）

**什么是"砖"？**
- **软砖**：系统起不来但 U-Boot 还在 → 按住 Reset 通电进恢复模式重刷 → 能救
- **硬砖**：BL2 或 FIP 坏了 → 需要拆机上编程器 → 非常难救

**保命法则**：刷机只动 mtd8（内核）和 mtd9（用户数据），千万不要碰 mtd0-mtd4！

<a name="append-a-file"></a>
### A.3 路由器的文件系统——什么目录放什么东西

> 看懂这个，你就能理解教程里提到的 `/etc/config/`、`/tmp/`、`/overlay` 都是什么。

```
路由器文件系统 (/)
│
├── /etc/                         ← 所有配置文件的"家"
│   ├── /etc/config/              ← UCI 配置（最常改的目录）
│   │   ├── network               ← 网络接口、IP、DHCP
│   │   ├── wireless              ← WiFi 设置
│   │   ├── firewall              ← 防火墙规则
│   │   └── dhcp                  ← DNS/DHCP 设置
│   ├── /etc/init.d/              ← 开机启动脚本
│   │   ├── network               ← 网络服务
│   │   ├── campus_auth           ← 我们的认证启动脚本
│   │   └── fast_recovery         ← 守护进程启动脚本
│   ├── /etc/crontabs/            ← 定时任务
│   │   └── root                  ← root 用户的定时任务表
│   └── /etc/dropbear/            ← SSH 服务器配置
│
├── /etc/campus_network/          ← 我们的脚本（自己创建的目录）
│   ├── auto_login.sh             ← 认证脚本
│   ├── fast_recovery_daemon.sh   ← 守护进程脚本
│   ├── route_watchdog.sh         ← 路由看门狗（已被守护进程替代）
│   └── dns_fix.sh                ← DNS 修复脚本
│
├── /tmp/                         ← 临时文件（重启后清空！）
│   └── campus_network.log        ← 认证日志
│
├── /rom/                         ← 出厂固件（只读，不要改）
├── /overlay/                     ← 你的修改的实际存储位置
├── /usr/bin/                     ← 可执行程序
├── /www/                         ← LuCI Web 界面
│
├── /proc/                        ← 虚拟文件系统（内核信息，实时生成）
│   └── /proc/mtd                 ← 闪存分区信息
│
└── /sys/class/net/               ← 网络接口信息
```

**文件后缀说明**：
| 后缀 | 是什么 | 例 |
|------|--------|-----|
| `.sh` | Shell 脚本（可执行） | `auto_login.sh` |
| 无后缀 | UCI 配置文件（在 /etc/config/ 下） | `network`, `wireless` |
| `.conf` | 传统配置文件 | `dnsmasq.conf` |

### A.4 看门狗和守护进程是什么？

**看门狗（Watchdog）**：一种自动监控机制。它定期检查"系统是否正常"，不正常就自动修复。

名字来源：硬件看门狗是一个定时器——系统需要定期"喂狗"（重置定时器），如果系统卡死了没喂，看门狗就自动重启系统。

本项目的软件看门狗：
```
每 N 秒检查："默认路由还在吗？"
  ├── 在 → 正常，继续监控
  └── 不在 → 紧急恢复！DHCP 续租 / WiFi 重连
```

**守护进程（Daemon）**：一种在后台持续运行的程序。不直接和用户交互，默默工作。

我们的 `fast_recovery_daemon.sh` 就是一个守护进程——它每秒检查一次网络，断了就立即修复。比 cron 定时任务（最小间隔 1 分钟）快得多。

**我们为什么从 cron 看门狗升级到守护进程？**
- Phase 7 (cron 看门狗)：每 20 秒检查 → 最坏情况 20s 才发现断网 + 8s wifi reload = 28s 断网
- Phase 10 (守护进程)：每秒检查 → 最坏情况 1s 发现 + 3s 恢复 = 4s 断网
- 提升：**7 倍**

```
radio0 (2.4G) = AP 模式
    发射 SSID "你的WiFi名"
    宿舍设备连接

## 附录 B：常见问题 FAQ

### Q1：刷机中途断电了，路由器现在没反应了怎么办？

> 如果 U-Boot 还在（mtd0-mtd4 没损坏）：按住 Reset 键通电，进入 U-Boot 恢复模式，通过 TFTP 重新刷入固件。
>
> 如果是"硬砖"（BL2 坏了）：需要用 TTL 串口和编程器修复。**只要你不去动 mtd0-mtd4，这种情况几乎不会发生**。

### Q2：为什么我的 WiFi 信号比原厂固件差？

> 可能 Factory 分区（WiFi 校准数据）损坏了。检查 `step1解锁ssh/xmir-patcher-main/backups/` 里的备份还在不在。如果丢了，只能用别人的同型号备份（信号也会差一些）。

### Q3：认证脚本运行失败，日志显示连不上认证服务器？

> 检查 STA 连接是否正常：`iw dev phy1-sta0 link`
> 检查 IP 是否拿到：`ip addr show dev phy1-sta0`
> 如果 STA 断了，可能是校园网 AP 踢了你。`wifi reload` 重新连接。

### Q4：手机连 WiFi 显示"无互联网"但实际能上网？

> 这是手机系统的误报。手机通过访问特定 URL 判断网络状态，校园网有时会干扰这个检测。以浏览器实测为准。

### Q5：校园网是 WPA-Enterprise（需要输用户名密码的那种 WiFi），不是开放 Portal，怎么处理？

> 需要安装 `wpa-supplicant`，配置 `/etc/wpa_supplicant.conf`，使用 802.1X 认证（PEAP/TTLS 等）。具体方法查阅你学校网络中心提供的 Linux 连接指南。

### Q6：我想换回小米原厂系统怎么办？

> 使用教程文件夹中的 `恢复原厂系统/` 工具。里面有小米官方固件和修复工具。

<a name="appendix-c"></a>
## 附录 C：拓展玩法

> 路由器不只是"上网工具"。刷了 OpenWrt 后，它是一台真正的 Linux 服务器。
> 但 AX3000T 有硬件限制（256MB RAM、无 USB 口），不是所有玩法都适合。
> 以下按实用性排序，标注了硬件兼容性和真实价值。

### ✅ 强烈推荐（立刻值得做）

#### 1. 🛡️ 广告拦截 (AdBlock) — ⭐⭐⭐

**效果**：所有连路由器的设备都不显示广告（视频 App 内广告除外）。

```bash
opkg update
opkg install luci-app-adblock
# LuCI Web → Services → Adblock → 启用
```

**原理**：DNS 层面拦截。广告域名的 DNS 请求返回 `0.0.0.0`，广告加载失败但不影响正常网页。这是 Pi-hole 的 OpenWrt 平替。

#### 2. 📊 流量监控 (nlbwmon) — ⭐⭐⭐ ✅ 已启用

**效果**：看到每个设备用了多少流量，谁在疯狂下载一目了然。

```bash
opkg install luci-app-nlbwmon
# LuCI Web → Services → Bandwidth Monitor
```

> 已配置：30 秒刷新，24 小时提交数据库，每月 1 日重置统计。数据库保留最近 10 个月。

#### 3. 🔒 DNS over HTTPS — ⭐⭐⭐

**效果**：防止校园网/运营商劫持你的 DNS 查询（比如把错误的淘宝网址塞给你）。

```bash
opkg install https-dns-proxy luci-app-https-dns-proxy
```

### ⚠️ 有条件推荐（特定场景有用）

#### 4. 🔑 VPN 客户端 (WireGuard) — ⭐⭐⭐ ✅ 已启用

**效果**：路由器作为 VPN 客户端接入远程网络，宿舍所有设备无需安装客户端即可访问 VPN 内部资源（远程桌面、内网服务等）。

```bash
opkg install kmod-wireguard wireguard-tools luci-app-wireguard
# 配置: wg0 接口, 分流模式 (仅 VPN 子网走隧道)
# LuCI: Network → Interfaces → wg0
```

> **已配置**：分流模式（`allowed_ips=10.66.66.0/24`），VPN 流量走隧道，正常上网仍走校园网直连。25 秒 keepalive 保活，穿透校园网 NAT。
> WireGuard 性能：在 MT7981B 上约 300-500 Mbps。校园网出口只有 100Mbps，CPU 完全不是瓶颈。
> **注意**：此处配置的是 VPN **客户端**（接入远程服务器），不是服务器（让外部连入）。校园网给 10.x.x.x 私网 IP，路由器无法作为 VPN 服务器被外部主动连接。但作为客户端主动连出是可以的。

#### 5. 🏷️ DDNS 动态域名 — ⭐⭐

**效果**：用域名（如 `my-dorm.duckdns.org`）访问路由器，不用记 IP。

> **限制**：校园网给的是私网 IP，DDNS 对校外访问没有帮助。但如果配合 VPN + 校内可路由，有一定价值。

### ❌ 不推荐（硬件限制或场景不匹配）

#### 6. 📁 文件共享 NAS — ❌

**不可行**。AX3000T **没有 USB 接口**，无法连接 U 盘或移动硬盘。没有 USB = 没有 NAS。

> 如果你需要宿舍 NAS，考虑：① 换带 USB 口的路由器（如 AX6000）；② 用一台旧电脑/树莓派做 NAS。

#### 7. 🖨️ 打印机服务器 — ❌

**不可行**。同上，需要 USB 口连接打印机。

#### 8. 🎬 DLNA 媒体服务器 — ❌

**不推荐**。虽然可以装 `minidlna`，但 overlay 只剩约 58MB 空间，存不了什么媒体文件。没有 USB = 没有大存储。

#### 9. 🐳 Docker 容器 — ❌

**不推荐**。256MB RAM 太小，OpenWrt 自己就用掉约 130MB，剩下不够跑 Docker。Docker 需要至少 512MB-1GB 才能实用。

#### 10. 🌐 透明代理/软路由 — ⚠️

**技术上可行但不推荐**。256MB RAM 跑代理客户端 + 分流规则很紧张。而且：
- 校园网出口只有 100Mbps，直连够用
- 透明代理配置复杂，出问题整个宿舍断网
- **法律提醒**：合理使用，遵守法规

#### 11. 👥 访客网络 — ⭐⭐

**可行但需求不大**。宿舍场景下不太需要"隔离访客 WiFi"。

#### 12. 🌍 IPv6 公网访问 — ⭐

**看校园网分配**。如果校园网分配了公网 IPv6，可以用。但很多校园网的 IPv6 也是私网的。

### 📊 资源利用建议

你的路由器资源状态（典型）：
```
CPU:  双核 A53 1.3GHz — 日常 < 5%
RAM:  256MB — 已用 ~130MB，空闲 ~126MB
Flash: 78MB overlay — 已用 ~20MB，空闲 ~58MB
```

**还能做什么**：
- ✅ 装更多 opkg 包（58MB 空闲可以装很多）
- ✅ 跑轻量脚本和服务（CPU/RAM 都够）
- ✅ 学 Linux（SSH 进去就是真 Linux 环境）
- ❌ 大存储（无 USB，58MB overlay 是全部空间）
- ❌ Docker（256MB RAM 不够）
- ❌ 高负载 VPN（CPU 会被吃满）

> 💡 **最佳利用方向**：把它当作**学习 Linux/网络的实验平台** + **宿舍网络管家**。写脚本、学防火墙、搭 Web 服务——这些才是它真正擅长的事。
