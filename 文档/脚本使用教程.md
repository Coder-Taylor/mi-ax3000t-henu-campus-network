# AX3000T 一键配置脚本使用教程

> 适用：已刷好 ImmortalWrt，需要快速配置校园网自动上网的人。
> 耗时：5-10 分钟（交互式，跟着提示走）。

---

## 第一步：物理连接

```
   ┌──────────┐                         ┌──────────────────┐
   │  你的电脑  │──── 网线 ──────────────→│  AX3000T 路由器    │
   │          │   插路由器的 LAN4 口       │  只插电源线+网线   │
   └──────────┘                         └──────────────────┘
```

1. 路由器通电开机，等 2 分钟
2. 电脑网线接路由器 **LAN4** 口（注意不要插 WAN/LAN1）
3. 电脑浏览器打开 `http://192.168.1.1`，确认能看到 LuCI 登录页
4. 确认能 SSH 连上：`ssh root@192.168.1.1`（首次无密码）

---

## 第二步：上传脚本到路由器

在电脑上打开终端（Git Bash / PowerShell / CMD），执行：

```bash
# 如果脚本在你电脑的下载目录
scp auto_config.sh root@192.168.1.1:/tmp/
```

如果提示输入密码，第一次是空密码（直接回车）。如果你之前设过密码就输密码。

---

## 第三步：运行脚本

```bash
ssh root@192.168.1.1
sh /tmp/auto_config.sh
```

脚本会分 7 步引导你完成所有配置，**每一步都有确认提示**。

---

## 脚本流程详解

### Step 0：环境检查
- 自动检测是否在 OpenWrt 系统上
- 检查外网连接
- 确认 overlay 空间充足

### Step 1：设置 root 密码
- 输入两次确认
- 保护 SSH 和 Web 后台

### Step 2：配置宿舍 WiFi
- 输入 WiFi 名称（支持中文）
- 输入 WiFi 密码
- 配置在 2.4G 频段（穿墙好）

### Step 3：配置无线 WAN
- 输入校园网 WiFi 名称（如 `henu-student`）
- 选择是否有 WiFi 密码（校园网通常是开放网络）
- 自动把 5G radio 改为 STA 模式
- 自动创建 wwan 接口 + 配置防火墙
- **这一步完成后路由器已经连上了校园网**

### Step 4：部署认证脚本
- 输入学号、校园网密码、运营商
- 脚本自动生成并部署到路由器
- **立即测试认证**——如果能看到"认证成功"，外网就通了

### Step 5：定时重连
- 自动添加 cron 每 5 分钟检测
- 网络断了自动重连

### Step 6：CAKE 智能流控
- 安装 sqm-scripts
- 配置 CAKE（110M 上限，平时不限速）
- triple-isolate：按设备公平分配
- 游戏/视频电话永远不卡

### Step 7：Argon 主题
- 安装 Argon 美化 LuCI
- 可换壁纸、暗色模式

---

## 第四步：验证效果

脚本跑完后：

```bash
# 1. 检查校园网连接
iw dev phy1-sta0 link
# 应该看到 Connected to henu-student

# 2. 检查 IP
ip addr show dev phy1-sta0 | grep inet
# 应该看到 10.x.x.x 的校园网IP

# 3. 检查外网
ping 8.8.8.8
# 应该 ping 通

# 4. 检查 CAKE
tc -s qdisc show dev phy1-sta0 | head -3
# 应该看到 cake 和 bandwidth 110Mbit
```

## 第五步：物理部署

配置完成后：

```
  墙壁电源
      │
      ▼
  ┌──────────────┐
  │  AX3000T     │  ← 只插电源线！
  │  无需任何网线 │
  └──────────────┘
      │
      │ 5G STA → henu-student (校园网WAN)
      │ 2.4G AP → 宿舍WiFi热点
      ▼
  全宿舍设备连 WiFi 上网
```

**拔掉所有网线，只留电源线。** 把路由器放在宿舍信号最好的位置（靠近窗户/门口通常校园网信号最好）。

## 常见问题

| 问题 | 解决 |
|------|------|
| 认证失败 | `cat /tmp/campus_network.log` 查看哪一步出错 |
| 上不了网 | 先 `ping 8.8.8.8`，不通就手动 `/etc/campus_network/auto_login.sh` |
| WiFi 搜不到 | `wifi reload` |
| 想改密码 | `passwd` 或 LuCI Web 后台改 |
| 想改 WiFi | LuCI → 网络 → 无线 |
| 想恢复出厂 | `firstboot` 然后重启 |

## 进阶

脚本跑完后你还可以去 LuCI Web 后台：

- **状态 → 实时流量**：看谁在用带宽

## 网络延迟监控工具

配置完成后，可以用以下工具持续监控校园网质量。
工具放在 `监控工具/` 文件夹下。

### 快速开始

**双击 `监控工具/启动监控.bat`**，选择要运行的工具即可。

### 工具清单

| 文件 | 用途 | 运行时间 |
|------|------|---------|
| `启动监控.bat` | 菜单入口，双击打开 | - |
| `校园网延迟监控.ps1` | 实时监控（14站点，0.5秒刷新，折线图） | Ctrl+C 停止 |
| `校园网压力测试.ps1` | 压力测试（14站点 × 30轮） | 约 5 分钟 |
| `校园网延迟对比.ps1` | 对比两次测试结果，看变化趋势 | 即时 |
| `测试记录/` | 自动保存的JSON测试数据 | - |

### 实时监控

双击 `启动监控.bat` 选 [1]，或在 PowerShell 中：
```powershell
cd 监控工具
.\校园网延迟监控.ps1
```

功能：
- 14 个站点同时监控（1 个内网网关 + 13 个外网站点）
- **0.5 秒刷新**，实时更新
- 每个站点显示中文名称（如 `百度  baidu.com`）
- 显示当前延迟、平均、最小、最大、丢包数
- **TCP 实时折线图**（最近 30 个采样点，上下波动）
- 按 Ctrl+C 停止后自动保存结果到 `测试记录/`

### 压力测试

双击 `启动监控.bat` 选 [2]，或在 PowerShell 中：
```powershell
cd 监控工具
.\校园网压力测试.ps1
```

功能：
- 14 站点 × 30 轮 = 420 次采样
- 实时进度条显示测试进度
- 测试完成后输出：TCP/SSL 统计表、柱状图、网络质量评估
- 自动保存结果到 `测试记录/`

### 对比测试

双击 `启动监控.bat` 选 [3]，或在 PowerShell 中：
```powershell
cd 监控工具
.\校园网延迟对比.ps1
```

功能：
- 自动列出所有测试记录
- 选择两次测试进行对比
- 显示每个站点的延迟变化（变快/变慢/持平）
- 柱状图直观对比

### 延迟等级说明

| 颜色 | TCP 延迟 | 含义 |
|------|---------|------|
| 🟢 绿色 | < 50ms | 正常 |
| 🟡 黄色 | 50-150ms | 偏慢 |
| 🔴 红色 | > 150ms | 很慢 |

### 前置要求

- Windows 10/11 自带 `curl.exe` 和 PowerShell
- 无需安装任何第三方程序
