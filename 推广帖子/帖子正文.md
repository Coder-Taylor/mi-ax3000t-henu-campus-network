#### Day 2 — 设备重连断网

**【3.png】**
> WiFi 连接详情 — 连上就能用

之前连过的手机重新连 WiFi 时，所有设备短暂断网。挂机游戏弹"网络异常"。

排查原因：部分设备连入时触发了芯片冲突，路由器被迫重启无线模块，全网断。

修了 6 个地方：

| 修复 | 效果 |
|------|------|
| 频宽对齐 | 减少不匹配导致的重启 |
| 信道锁定 | 不让它自动跳信道，消除周期性断连 |
| 路由看门狗 | 路由丢失后自动恢复 |
| 认证脚本升级 | 认证 + 路由恢复一体化 |
| 定时优化 | 快速恢复 + 低频认证，合理分工 |
| 开机自启 | 路由器通电就自动运行 |

**【4.png】**

#### Day 3 — 秒级恢复 + DNS 修复

**【5.png】**
> 流量消耗器 — 实测 102Mbps，12.8MB/s

把看门狗从定时检测改成每秒检测的守护进程。

踩了 9 个坑，逐个解决：

| 坑 | 解决 |
|----|------|
| 重载无线太重，会踢掉所有设备 | 改成只重连指定信道 |
| 进程检测不准 | 双重校验机制 |
| 守护进程挂了不会重启 | 用系统进程管理，自动恢复 |
| DNS 重启影响其他服务 | 加延迟保护 |
| 服务名冲突 | 重新定义标识 |
| 日志刷屏 | 关键事件才记录 |
| 启动顺序问题 | 调整优先级 |
| 多进程并发 | 加锁机制 |
| 信号量残留 | 启动时清理 |

最终：断网 1-2 秒自动恢复。

**【6.png】**
> 压力测试 — 14 站点 × 30 轮，全通过

同一天修了校园内部网站 DNS。

校园网的 DNS 对部分域名解析有问题，有的打不开。研究了半天，发现两个 DNS 服务器各有盲区。解决方案：不同域名走不同的 DNS，再加 hosts 文件写死。

#### 智能带宽分配

这套系统有一个关键能力：**多人共用时，不会一个人下东西全宿舍卡。**

原理：CAKE 流控 + ACK 过滤。

| 机制 | 作用 |
|------|------|
| 每设备公平分配 | 四台设备各拿 1/4 带宽，不会被一台设备抢光 |
| ACK 过滤 | 自动过滤冗余的确认包，优先保障游戏等小包传输 |
| 三重隔离 | 源、目的、流三维度隔离，互不干扰 |

实际效果：一个人跑满速下载，另外三个人打游戏、看视频不受影响。
